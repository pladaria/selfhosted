# Contributing

## Language Requirements

All contributions must be written in English. This includes:

- Source code
- Code comments
- Documentation
- Commit messages
- Pull request descriptions
- Issue reports

## Code Standards

- Write clear, descriptive variable and function names
- Add comments for complex logic or non-obvious implementations
- Follow existing code style and conventions
- Test your changes before submitting

## Documentation

- Update relevant documentation when adding or modifying features
- Keep README.md current with any new dependencies or usage changes
- Document command-line arguments and configuration options

## Submitting Changes

- Create a pull request with a clear description of the changes
- Reference any related issues
- Ensure code runs without errors
- Verify all existing functionality remains intact
